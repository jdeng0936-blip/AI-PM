# 徽远成 AI-PM（智能项目经理）— 白皮书·总体设计

> **版本**：v1.0 | **日期**：2026-03-05 | **来源**：基于 Gemini 共享文档提炼

---

## 一、项目愿景与背景

### 1.1 痛点分析

传统项目管理依赖 12 列 Excel 日报，存在以下核心问题：

| 痛点 | 表现 |
|------|------|
| 录入反人类 | 员工面对 12 个格子抓耳挠腮，日报敷衍、大量留白 |
| 检查靠人工 | 主管需逐条审阅，耗时且易遗漏逻辑漏洞 |
| 监督无连续性 | 跨日任务追踪断裂，卡点无人跟进 |
| 评价主观化 | 缺乏量化标准，考核失去公信力 |
| 信息碎片化 | CEO/CTO 需翻阅大量微信消息才能了解全局 |

### 1.2 系统定位

**"徽远成 AI-PM"** 是一套以 **大模型（Gemini）** 为核心引擎的智能项目管理系统，目标是：

> **让日报录入从 5 分钟缩短到 10 秒，让管理从"人盯人"升级为"AI 驱动的自动化 PDCA 闭环"。**

### 1.3 团队优势

- 团队已深度使用大模型（张毅做 NewApi/Liberchat 接入、郭震用 Antigravity 写代码、陈家云用智能体优化仓库）
- 具备 "Eat Your Own Dog Food" 的内部验证能力
- 技术栈前沿、软硬结合，可快速自研落地

## 二、核心功能模块设计

系统围绕 **PDCA（Plan-Do-Check-Act）** 管理闭环，设计了 **五大核心模块**：

```
┌─────────────────────────────────────────────────────┐
│                 AI-PM 系统架构总览                     │
├─────────────────────────────────────────────────────┤
│                                                     │
│   ① 录入(Input)  ──→  ② 检查(Check)  ──→            │
│        ↑                                            │
│        │           ③ 监督(Supervise)                  │
│        │                │                            │
│        └────────────────┘                            │
│                                                     │
│   ④ 评分(Score)    ⑤ 管理(Manage)                     │
│      ↓                  ↓                            │
│   游戏化激励          战情看板 + 自动周报               │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 模块 ① — 对话式录入 (Chat-to-Form)

**核心理念**：从"填 12 个格子"变成"发一条消息/语音"

| 要素 | 设计方案 |
|------|---------|
| **交互入口** | Liberchat 客户端 / 钉钉·企微机器人 |
| **输入方式** | 自然语言文字或语音（极简输入） |
| **AI 解析引擎** | Gemini 1.5 Flash — JSON 结构化输出 |
| **确认机制** | 系统弹出卡片 → 员工一键确认提交 |
| **效率提升** | 5 分钟 → 10 秒 |

**数据流**：

```
员工语音/文字 → Gemini 1.5 Flash 解析 → 结构化 JSON (12字段)
    → 卡片预览 → 员工一键确认 → 入库 (PostgreSQL)
```

**AI 自动填充字段示例**：

| 字段 | 自动提取示例 |
|------|------------|
| 姓名 | 张毅 |
| 今日任务 | 测试 vertex-ai 等渠道接入最新模型到 NewApi，测试接入 Liberchat |
| 完成进度 | 100% |
| 代码归档 | v2.1 |
| 验收标准 | NewApi 分发正常，Liberchat 端可正常对话（AI 自动补全） |

---

### 模块 ② — 智能质检 (Check)

**核心理念**："无情质检员" — 提交瞬间自动拦截逻辑漏洞

**质检规则**：

| 规则类型 | 检测逻辑 | 触发动作 |
|---------|---------|---------|
| **逻辑漏洞拦截** | 进度 < 100% 但【核心卡点】为空 | 打回要求补充 |
| **验收标准 SMART 审查** | 任务完成但验收标准为空 | 提示补充具体标准 |
| **规范校验** | 研发任务 100% 但无产出物（代码版本号/成果演示） | 要求补充 |

**拦截示例**：

> 👉 **张维**，进度 70%，但【核心卡点】和【解决方案】为空  
> → AI 提示：*"@张维，进度未达100%，请务必补充『核心卡点』及『预计解决时间』，以便团队了解阻碍并协调资源。"*

> 👉 **林跃文**，采购 MCU 任务，验收标准为空  
> → AI 提示：*"@跃文，采购任务不能没有验收标准哦，请补充是要求【下单完成】还是【物料入库清点完毕】？"*

---

### 模块 ③ — 跨日监督 (Supervise)

**核心理念**：AI 充当"不知疲倦的敏捷教练"，具备上下文理解能力

| 能力 | 描述 |
|------|------|
| **跨日连贯性监督** | 自动追踪前日卡点，标记为 `[待解决]`，次日重点监督推进情况 |
| **上下游依赖预警** | 识别任务间依赖关系，当上游阻塞时自动提醒下游调整节奏 |

**场景示例**：

- **跨日追踪**：陈家云 25 日提到"原材料仓缺实操"，AI 自动在 26 日监督该问题是否推进
- **依赖预警**：林跃文（采购 MCU/LoRa 模块）的进度直接影响 206 样机贴片与测试。若采购延迟，AI 自动给下游发预警

---

### 模块 ④ — 游戏化评分 (Score)

**核心理念**：多维度量化打分 + "反卷"提醒

**评分维度**：

| 维度 | 权重 | 说明 |
|------|------|------|
| 日报含金量 | 高 | 是否有实质产出描述 |
| 逻辑严密性 | 高 | 进度、卡点、方案是否自洽 |
| 规范完整度 | 中 | 必填字段是否齐全 |
| 协作贡献度 | 中 | 是否主动协调跨部门问题 |

**评分输出示例**：

```json
{
  "员工": "陈家云",
  "日期": "2月25日",
  "评分": 85,
  "等级": "良好",
  "AI点评": "工作态度积极，主动协调原材料问题，但在解决问题时效性上需在26日进一步明确节点",
  "推荐": "当日表现优秀员工",
  "管理警告": "仓库缺实操环境，可能导致后续流程阻塞，请主管郭震协助解决"
}
```

---

### 模块 ⑤ — 战情管理 (Manage)

**核心理念**：老板的"战情看板" — 每日 9:00 自动推送

| 功能 | 描述 |
|------|------|
| **项目健康度概览** | 🔴 红（卡点严重/延期）🟡 黄（依赖项异常）🟢 绿（正常推进）三色仪表盘 |
| **今日核心关注** | 全公司最紧迫的 Top 3 技术卡点 |
| **自动化周报** | 从日报中自动提炼关键里程碑 |

## 三、技术架构设计

### 3.1 整体架构

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   前端入口    │     │   后端中枢    │     │   AI 引擎    │
│              │     │              │     │              │
│ · Liberchat  │────▶│ · FastAPI    │────▶│ · Gemini     │
│ · 钉钉/企微   │     │ · Python     │     │   1.5 Flash  │
│   机器人      │     │              │     │ · NewApi     │
│              │     │ · PostgreSQL │     │   (模型分发)   │
└──────────────┘     └──────────────┘     └──────────────┘
```

### 3.2 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| **模型分发** | NewApi | 张毅已部署，统一管理多模型渠道 |
| **AI 模型** | Gemini 1.5 Flash | JSON 结构化输出、响应快、成本低 |
| **后端框架** | Python FastAPI | 业务逻辑中枢，JSON 解析与路由 |
| **数据存储** | PostgreSQL | 结构化日报数据持久化 |
| **前端入口** | Liberchat 插件 / 钉钉·企微 | 复用现有聊天基础设施 |

### 3.3 核心 Prompt 工程

系统的 AI 行为由精心设计的 System Prompt 驱动：

```python
system_prompt = """
你现在是徽远成科技的"AI 项目经理"。请根据以下员工提交的单日工作记录JSON，
进行严格的检查、指出问题，并给出一个0-100分的综合评分。

【检查规则】
1. 逻辑校验：如果"完成进度"不足100%，但"核心卡点"为"无"或空，
   判定为逻辑不合格(pass_check: false)，必须扣分。
2. 规范校验：研发任务进度达到100%时，必须检查是否有产出物
  （代码版本号或成果演示）。

【返回格式要求 (必须是严格的JSON)】
{
  "pass_check": true/false,
  "reject_reason": "如果不通过，请用温和的语气指出需要员工补充什么字段",
  "score": 85,
  "ai_comment": "作为主管的一段点评（50字以内，包含肯定与建议）",
  "management_alert": "如果有进度严重滞后或需要主管协调资源，
                       请在此输出警告提取，否则为空"
}
"""
```

## 六、总结

**AI-PM 系统的核心价值**：

1. ✅ **效率革命** — 日报录入从 5 分钟 → 10 秒

> **一句话总结：用 AI 将"每日苦不堪言的填表 + 主管逐条审阅"转变为"说句话就完事 + AI 全自动 PDCA 闭环管理"。**
